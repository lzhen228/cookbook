### 2.1 进入列表

点击左侧菜单「**供应商管理**」，即进入供应商列表页面。

### 2.2 页面功能说明

```

├─────────────────────────────────────────────────────┤
│               < 1  2  3  4  5 >            共 342 条 │

```

### 2.3 筛选供应商

#### 关键字搜索

在搜索框输入供应商名称关键字（支持模糊搜索），按 Enter 或等待 500ms 自动触发搜索。

> **示例：** 输入「华为」，可匹配「华为技术有限公司」「华为终端有限公司」等。

#### 按健康等级筛选

点击「健康等级」下拉菜单，可多选：
- 🔴 **高风险**：健康分 0–39，需立即关注
- 🟠 **需关注**：健康分 40–69，持续监控
- 🟢 **低风险**：健康分 70–100，状态良好

#### 按合作状态筛选

| 状态值 | 说明 |
|--------|------|
| 合作中 | 当前有效合作供应商 |
| 潜在 | 正在评估中 |
| 合格 | 已通过资质审核但未合作 |
| 黑名单 | 禁止合作 |
| 限制 | 有条件限制合作 |

### 2.4 排序

点击列标题可切换排序：

| 可排序列 | 说明 |
|---------|------|
| 健康分 | 默认升序（最高风险在前） |
| 供应商名称 | 按中文拼音排序 |
| 创建时间 | 按录入时间排序 |

### 2.5 关注供应商

点击操作列的「☆」图标，将供应商加入关注列表（显示为「★」）。关注后可在看板快速查看重点供应商动态。

> **权限要求：** 需要「风险分析师」及以上角色。

### 2.6 查看供应商详情

点击供应商名称（蓝色超链接），进入该供应商的画像详情页。

### 2.7 数据时效说明

列表中的健康分数据来自**每日 00:00–04:00 的批量评分任务**，实际显示的是 T+1 数据。
「更新时间」列显示最近一次评分更新的时间，可据此判断数据新鲜度。

### 3.3 基础信息卡片

| 字段 | 说明 |
|------|------|
| 统一信用代码 | 18 位社会信用代码，唯一标识 |
| 合作状态 | 与本公司的当前合作关系 |
| 所在地区 | 省/市 |
| 上市状态 | 已上市 / 未上市 |
| 企业性质 | 民营/外资/国有/合资 |
| 供应类目 | 主要供应的产品/服务类别 |
| 世界/中国 500 强 | 是否入围 |

### 3.4 健康评分卡片

#### 综合健康分

- 取值范围：**0–100 分**
- 计算公式：`100 - Σ(指标扣分 × 权重)`
- 若触发**红线指标**（如严重司法违规），总分直接归零

#### 近 7 天趋势

- 正值（绿色 ▲）：得分较 7 天前上升，风险改善
- 负值（红色 ▼）：得分较 7 天前下降，风险加剧
- `--`：无历史数据，尚未评分满 7 天

#### 维度得分说明

| 维度 | 数据来源 | 关注要点 |
|------|---------|---------|
| 司法合规 | 司法大数据 | 诉讼记录、执行信息、失信被执行人 |
| 财务健康 | ERP/财务系统 | 账款逾期、财务异常 |
| 信用状况 | 征信数据 | 信用评分、违规记录 |
| 税务合规 | 税务数据 | 纳税等级、欠税记录 |
| 经营稳定 | 工商数据 | 经营异常、人员变动 |

> **维度得分 < 40 分**时，建议重点关注该维度的具体风险事项。

### 3.5 Tab 数据区域（懒加载）

点击 Tab 标签后，系统将从对应数据源拉取数据（**首次加载约 1–3 秒**）。

| Tab | 内容 |
|-----|------|
| 基本信息 | 注册信息、法人代表、经营范围、股东信息 |
| 经营信息 | 工商年报、分支机构、股权结构变化 |
| 司法诉讼 | 近期诉讼记录、法院判决、执行信息 |
| 信用数据 | 信用评分历史、违规处罚记录 |
| 税务信息 | 纳税人等级（A/B/C/D）、欠税记录、税务异常 |

**数据陈旧提示：** 若 Tab 顶部出现黄色提示条「⚠ 数据来自缓存，可能已过期」，表示外部数据源当前不可用，显示的是最近一次成功获取的数据，请以实际时间为准。

### 3.6 下载供应商报告

1. 点击页面右上角「**下载报告**」按钮
2. 系统异步生成 PDF 报告（约 30 秒）
3. 生成完成后，点击链接下载（有效期 **15 分钟**）
4. 超过 15 分钟后链接失效，重新点击「下载报告」获取新链接

> **权限要求：** 需要「风险分析师」及以上角色。

点击左侧菜单「**风险预警**」，进入风险事项列表。

> 若菜单图标上有红色数字气泡，表示当前有未处理的风险事项。

### 4.2 页面功能

```
┌─────────────────────────────────────────────────────────────┐
│  今日新增: 5条    总计: 128条                                  │
│  [状态▼]  [风险维度▼]  [供应商▼]  [日期范围]                 │
├─────────┬──────────┬──────────────────────┬──────┬──────────┤
│ 供应商   │ 风险维度  │ 描述                  │ 状态  │ 操作     │
├─────────┼──────────┼──────────────────────┼──────┼──────────┤
│ 华为技术 │ 司法合规  │ 存在未结清司法判决…   │ 待处理│ [处理]   │
│ 比亚迪   │ 财务健康  │ 连续3个月账款逾期…    │ 处理中│ [查看]   │
└─────────┴──────────┴──────────────────────┴──────┴──────────┘
```

### 4.3 风险事项状态流转

```
[待处理] ──→ [已确认] ──→ [处理中] ──→ [已关闭]
    └───────────────────────────────→ [已忽略]
```

| 状态 | 说明 | 可操作至 |
|------|------|---------|
| **待处理** | 系统自动生成，尚未人工确认 | 已确认、已忽略 |
| **已确认** | 人工确认确实存在此风险 | 处理中、已忽略 |
| **处理中** | 已指派负责人，正在跟进 | 已关闭 |
| **已关闭** | 风险已消除，填写关闭说明 | — |
| **已忽略** | 经评估认为无需处理 | — |

### 4.4 处理风险事项

**步骤：**

1. 在列表中找到目标风险事项，点击「处理」按钮
2. 查看事项详情，确认风险属实后选择「确认」
3. 指派负责人（可选：自己或其他团队成员）
4. 更新状态为「处理中」，并记录处理进展
5. 风险消除后，填写关闭说明，将状态更新为「已关闭」

> **权限要求：** 更改事项状态需要「风险分析师」及以上角色。

### 4.5 筛选与导出

| 筛选条件 | 说明 |
|---------|------|
| 状态 | 按处理状态筛选 |
| 风险维度 | 按司法/财务/信用/税务/经营维度筛选 |
| 供应商 | 查看指定供应商的所有风险事项 |
| 日期范围 | 按风险触发日期范围筛选 |

## 5. 预警方案管理（管理员）

> **权限要求：** 本章功能仅「风险管理员」可操作。

### 5.1 什么是预警方案？

预警方案定义了供应商健康分的计算规则，包含：
- 各风险指标的**权重**（所有指标权重之和必须 = 100%）
- 哪些指标为**红线指标**（触发即总分归零）
- 各健康等级的**分值阈值**

### 5.2 查看现有方案

点击左侧菜单「**预警方案**」，查看所有方案列表。

- 🟢 **激活中**：当前正在使用的评分规则，每日批量评分依据此方案计算
- ⚪ **未激活**：已配置但未启用的方案

### 5.3 创建新方案

**步骤：**

1. 点击「**新建方案**」按钮
2. 填写方案名称（最多 30 字）和描述
3. 配置**指标权重**：
   - 从指标库选择指标
   - 为每个指标设置权重（0–100%）
   - **所有指标权重之和必须等于 100%**，否则无法保存
   - 勾选「红线指标」表示该指标触发后总分直接归零
4. 配置**健康等级阈值**（可使用默认值：70以上低风险，40-70需关注，40以下高风险）
5. 点击「**保存**」

> **提示：** 新建方案默认为「未激活」状态，不影响当前评分结果。

### 5.4 激活方案

1. 在方案列表找到目标方案，点击「**激活**」
2. 确认弹窗提示（当前激活方案将自动停用）
3. 激活成功后，**下一次批量评分任务**（次日 00:00）将使用新方案

> **注意：** 激活操作不可撤销，请在非业务高峰期操作。

### 5.5 注意事项

- **激活中的方案不可删除**，需先激活另一方案后再删除
- 修改已激活方案的权重配置，需要创建新方案并激活，旧方案保留用于历史数据对比
- 指标权重变更后，历史快照数据不会回算

## 6. 常见问题 FAQ

### Q1：供应商健康分为什么是昨天的数据？

**答：** 平台采用每日批量计算模式，每天 00:00 开始对全量供应商重新评分，04:00 前完成。因此列表页展示的是 T+1 数据（前一天的评分结果）。若需要查看实时数据，请查看画像页的「数据更新时间」字段。

---

### Q2：为什么某供应商健康分突然变为 0？

**答：** 该供应商触发了**红线指标**（如被列为失信被执行人、涉及重大刑事案件等）。触发红线指标时，系统强制将总分置为 0，不论其他维度表现如何。请及时查看该供应商的风险事项，了解具体触发原因。

---

### Q3：Tab 数据一直显示「加载中」或「暂无数据」？

**答：** 可能原因：
1. **外部数据源暂时不可用**：等待约 10–30 分钟后刷新重试
2. **该供应商数据尚未获取**：系统首次采集需要时间，通常 24 小时内完成
3. **网络问题**：检查网络连接后刷新页面

若 Tab 顶部出现黄色提示「数据来自缓存」，说明当前显示的是历史缓存数据，外部数据源暂时不可用，属于系统降级保护机制。

---

### Q4：报告下载链接为什么失效了？

**答：** 安全策略要求，报告下载链接有效期为 **15 分钟**。超过后请返回画像页，重新点击「下载报告」获取新的下载链接。

---

### Q5：我想关注某个供应商，但「关注」按钮是灰色的？

**答：** 关注功能需要「风险分析师」或「风险管理员」角色，「查阅者」角色无此权限。如需申请权限提升，请联系系统管理员。

---

### Q6：为什么搜索时找不到某个供应商？

**答：** 可能原因：
1. 该供应商尚未录入平台
2. 名称存在细微差异（如全称 vs 简称），尝试输入更短的关键字
3. 当前存在筛选条件过滤了该供应商，请清除所有筛选后重新搜索

---

### Q7：风险事项通知怎么接收？

**答：** 平台通过以下渠道推送预警通知：
- **企微机器人**：风险触发后 5 分钟内发送到指定群组
- **邮件通知**：发送至系统注册邮箱

收不到通知请联系管理员确认通知渠道配置。

---

### Q8：如何查看某供应商的历史健康分趋势？

**答：** 在供应商画像页，健康评分卡片显示近 7 天趋势变化量。完整历史趋势折线图（近 30 天）在「基本信息」Tab 中查看，或在风险看板全局趋势图中筛选单个供应商。

## 7. 名词解释

| 名词 | 解释 |
|------|------|
| **健康分** | 0–100 分的综合风险评分，分越低风险越高 |
| **红线指标** | 触发即导致健康分直接归零的严重风险指标 |
| **预警方案** | 定义评分规则的配置，包含各指标权重和健康等级阈值 |
| **游标分页** | 平台使用的高性能分页技术，基于上一页最后一条记录定位，性能稳定 |
| **Tab 懒加载** | 画像页 Tab 数据按需加载，点击 Tab 时才向服务器请求，减少首屏等待 |
| **预签名 URL** | 有时效性的文件下载链接，15 分钟后自动失效，保障报告安全 |
| **T+1 数据** | 数据延迟一天，今天看到的是昨天计算的结果 |
| **统一信用代码** | 企业的 18 位社会信用代码，全国唯一标识 |
| **司法合规维度** | 基于法院诉讼、执行信息、失信记录等计算的风险维度得分 |
| **数据降级** | 外部数据源不可用时，系统自动返回缓存历史数据，并标注数据时效 |
| **DLQ** | 死信队列，Kafka 消息消费失败超过重试次数后存入，防止消息丢失 |
